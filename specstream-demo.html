SpecStream Technical System Manual

1. The Problem Statement

Standard PDF text extraction reads character streams in raw flow order. When encountering multi-column text blocks or sidebar specs, the stream bleeds left-to-right across columns, destroying logical reading sequence. Standard extractors also strip all semantic layout hierarchies.

2. Dynamic XY-Cut Partitioning

SpecStream resolves multi-column bleeding by executing a recursive spatial division algorithm based on projection profiles. Horizontal whitespace gaps split the page into top/bottom bands; vertical gaps split into left/right columns. The algorithm recurses until leaf nodes are established and ordered top-to-bottom, left-to-right.

Toggle Naive mode in the header to compare how a linear top-to-bottom parser bleeds the two specification columns together.

3. Semantic Classification

Each block is classified into one of five types:

  • heading — Numbered section titles (1., 1.1., etc.)
  • warning — WARNING / CAUTION / NOTE blocks
  • footnote — Small-font annotations at page foot
  • table — Pipe-delimited tabular rows
  • spec — Body text and specification lines (fallback)

4. Tree Hierarchy Reconstruction

A stack-based assembler iterates the ordered classified blocks to construct a nested parent-child document tree. Headings act as parent nodes for their subsequent content blocks. The resulting hierarchy is shown in the sidebar and exported as JSON.

5. Controls

  • Sample: Load the built-in MCU datasheet demo.
  • Upload PDF: Parse any client-side PDF (single page, heuristic extraction).
  • XY-Cut / Naive: Toggle reading-order algorithm in real time.
  • Bounding boxes: Hover for type-colored glow. Click to select and inspect.
  • JSON Export: Expand the strip at the bottom to view or copy the full hierarchy JSON.
