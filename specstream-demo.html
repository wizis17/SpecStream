📐 SpecStream Technical System Manual

        
SpecStream is a layout-aware PDF document parser designed specifically for technical datasheets, blueprints, and engineering spec sheets.

        
        
1. The Problem Statement

        
Standard PDF text extraction reads character streams in raw flow order. When encountering multi-column text blocks or sidebar specs, the stream bleeds left-to-right across columns, resulting in interleaved text that completely destroys the logical reading sequence. Standard extractors also strip all semantic layout hierarchies, converting headings, tables, footnotes, and warnings into flat, undifferentiated lines.

        
        
2. Dynamic XY-Cut Partitioning Algorithm

        
SpecStream resolves multi-column bleeding by executing a recursive spatial division algorithm based on projection profiles:

        
    
          
  • Horizontal Projection Profiles: The engine merges all bounding box y-intervals across the page to find continuous horizontal whitespace gaps larger than the gap threshold (H_GAP). If found, the page is divided into top and bottom bands, and the parser recurses on each band.
    • 
          
  • Vertical Projection Profiles: If no horizontal cut is possible, the engine merges the x-intervals to find vertical whitespace columns larger than the gap threshold (V_GAP). If found, it partitions the space into left and right columns and recurses.
    • 
          
  • Fallback: When no cuts can be made, a leaf node is established. The remaining blocks within that leaf are ordered by top-to-bottom and left-to-right positioning.
    • 
          
  • Dynamic Gap Sizing: Rather than hardcoding fixed pixel gaps, SpecStream dynamically computes threshold gaps by analyzing the median line height of text components within the document.
    • 
        

        
        
3. Heuristic Semantic Classification

        
Once a sequence of blocks is ordered, each block is analyzed and tagged with a semantic block category:

        
    
          
  • title: Main document title (typically large font size 18px+).
    • 
          
  • subtitle: Auxiliary titles directly below headers.
    • 
          
  • heading1: Major sections (matched using section-numbering regex, e.g., 1.0, 2.0).
    • 
          
  • heading2: Subsections (e.g., 1.1, 2.1.3).
    • 
          
  • warning: Highlighted cautionary details matching alert keywords (WARNING, CAUTION, NOTE:, ⚠).
    • 
          
  • footnote: Metadata or references located at the page foot with small font sizes.
    • 
          
  • table-header & table-row: Structured blocks containing column delimiters (|) or uniform spatial tabulations.
    • 
          
  • spec: Technical datasheet spec line fallback.
    • 
        


        
4. Tree Hierarchy Reconstruction

        
A stack-based assembler iterates over the ordered classified blocks to construct a nested parent-child document tree. Headings act as parent blocks, structural annotations parent their adjacent content blocks, and tables/warnings partition logically. The resulting hierarchy tree is outputted as a standard JSON outline, which can be downloaded directly from the JSON tab in the sidebar.


        
5. Interface Controls

        
    
          
  • Upload PDF: Parse layouts from any client-uploaded datasheet dynamically.
    • 
          
  • Load Sample: Render the built-in blueprint sample document to test structural features.
    • 
          
  • Naive vs. XY-Cut Toggles: Compare how standard linear streams stack up against structured layout partitions.
    • 
          
  • Legend Badges: Check/uncheck categories to filter out specific bounding box overlays on the page viewer canvas.
    • 
          
  • Bidirectional Highlighting: Hover/click elements in the page viewer, hierarchy tree, or reading order list to highlight sync highlights across panels.
